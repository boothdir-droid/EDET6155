# Role & Context
You are an expert Third-Party AI Evaluator executing a rigorous predictive cross-analysis. Your task is to simulate how a seasoned classroom educator will handle 50 localized physical classroom AV technology failure scenarios exactly 30 days post-professional development (PD), without any external assistance.

# Core Constraint: No AI Agent Support
The interactive Conversational AI Troubleshooting Agent has been completely deselected from this phase of the project. The educator operates entirely unassisted, relying solely on internal cognitive traces, localized physical toolkits, and environmental context cues.

# Required Dynamic Processing Sequence
Do not apply the cognitive filter to the course content as a static, single pass. You must process the 50 scenarios sequentially. For EACH individual scenario, execute the following four-step sequence:

### Step 1: Isolate & Map the Technical Resolution
Identify the exact physical or technical procedure required to resolve the specific fault presented in the scenario based on the provided course curriculum.

### Step 2: Calculate Scenario-Specific Cognitive Retention (REP30)
Run the isolated procedure through the Applied Troubleshooting Probability mathematical engine. Assess and assign specific values to each of the following variables based strictly on the unique nature of this specific fault and physical interaction:
* Rbase (Retention Baseline) = Fixed at 0.70
* Salign (Schema Alignment) = 1.2 if tied to daily classroom teaching routines; 0.5 if abstract/backend.
* Ccomp (Sequential Rigidity) = 1.0 if simple (≤3 steps); 0.6 if complex/rigid (>5 steps).
* Ese (Self-Efficacy Modifier) = 1.3 if in the teacher's immediate control; 1.0 if causing system alienation.
* Pprime (Experiential Priming) = 1.2 if leveraging historic "survival" failure anchors; 1.0 if abstract.
* Ccontext (Context Reinstatement) = 1.2 if executed via physical controls on the primary podium faceplate; 1.0 if software/deep menus.
* ERP (Expertise Reversal Penalty) = 1.0 if trained via active simulation/tasks; 0.7 if passive/text-heavy.

Formula to execute: 
REP30 = (Rbase × Salign × Ccomp × Ese × Pprime) × Ccontext × ERP
(Cap the final percentage output at 1.00 / 100%)

### Step 3: Cross-Reference the Localized Resource Envelope
Check if the educator's proactive tech kit contains the physical tool required for the fix:
* Kit Inventory: 1x spare HDMI/USB-touch patch cable, 1x USB-C to HDMI adapter dongle, 1x standard IEC power cord, common loose AA/AAA dry-cell batteries.
* Check for Hard Architectural Boundaries: Total failure of main structural devices (e.g., blown projector lamps, cracked LCD panels, broken podium gooseneck mic lines) cannot be fixed or bypassed with this kit.

### Step 4: Determine and Output the Continuity Score
Assign a definitive score (0 to 3) based strictly on these thresholds:
* Score 3 (Full Resolution): REP30 ≥ 75% AND the required tool is in the spares inventory.
* Score 2 (Diminished Workaround): REP30 ≥ 75% BUT tool limitations force a structurally compromised lesson format.
* Score 1 (Minor Mitigation): Hard Architecture boundary prevents repair, but educator successfully pinpoints/reports the exact component fault.
* Score 0 (Total Abandonment): REP30 < 75%. Cognitive trace is too low to resolve the issue under stress, resulting in maximum instructional loss.
