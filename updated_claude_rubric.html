Role & Task Definition

Role: Expert Third-Party Evaluator

Task: Execute a predictive cross-analysis between provided training curriculum assets and a locked dataset of 50 localized classroom AV technology failure modes. The simulation maps whether a seasoned classroom educator can mitigate real-time faults 30 days post-training relying strictly on internal cognitive traces, localized physical toolkits, and context cues.

Part 1. Human Cognitive Architecture ($REP_{30}$ Memory Matrix)

Rather than treating learning loss as a flat, uniform drop-off, knowledge retention is processed through a multi-variable math profile modeling a seasoned educator at a 30-day post-PD interval. Evaluators must gauge cognitive capability using the Applied Troubleshooting Probability ($REP_{30}$) Formula:

REP30 = (Rbase × Salign × Ccomp × Ese × Pprime) × Ccontext × ERP
Variable & Baseline Cognitive Multipliers Evaluation Filter Criteria
$R_{base}$ (Retention Baseline)
Standard value = 0.70		 Fixed Constant The default memory trace remaining 30 days post-training before non-use decay accelerates.
$S_{align}$ (Schema Alignment) • High: 1.2
• Low/Abstract: 0.5		 Measures if the technical troubleshooting steps tie cleanly directly into familiar, daily classroom teaching routines.
$C_{comp}$ (Sequential Rigidity) • Simple (≤3 steps): 1.0
• Rigid (>5 steps): 0.6		 Account for complexity. High sequential rigidity or high step counts drop memory performance under stress.
$E_{se}$ (Self-Efficacy Modifier) • In Control: 1.3
• System Alienation: 1.0		 High professional self-efficacy acts as a protective buffer, preventing prefrontal cortex deactivation under stress.
$P_{prime}$ (Experiential Priming) • Salient Anchor: 1.2
• Abstract Context: 1.0		 Leverages prior episodic "survival" memories of historical technology failures (e.g., historical mid-lesson screen blackouts).
$C_{context}$ (Context Reinstatement) • Podium Interface: 1.2
• Deep Menu/Software: 1.0		 Physical cues (podium faceplate, cords, wall plates) serving as external environmental retrieval triggers.
$ERP$ (Expertise Reversal) • Applied Task/Sim: 1.0
• Text-Heavy/Hand-held: 0.7		 Penalizes passive, overly hand-held, or text-heavy explanations that cause cognitive load track mismatches.

*Note: A calculated $REP_{30}$ score ≥ 75% indicates a resilient applied capability that survives the 30-day non-use window. Cap final value at 1.00 (100%).

Part 2. Localized Resource Envelope

The simulation strictly limits outcomes to the local classroom environment. Because runtime AI troubleshooting support has been completely removed, the educator is entirely dependent on physical resources inside this boundary:

  • Fixed Architecture: Classrooms possess exactly one alternate auxiliary routing pathway (e.g., an available legacy VGA line if the HDMI line exhibits digital drop-out).
  • Proactive Tech Kit: The educator carries an inventory consisting strictly of: 1x spare digital patch cable (HDMI/USB touch-tether), 1x USB-C to HDMI adapter dongle, 1x standard IEC power cord, and a minimal pack of AA/AAA dry-cell batteries.
  • Hard Architectural Boundaries: Total failure of main structural devices (e.g., blown projector lamps, shattered LCD panels, broken built-in podium microphone lines) cannot be physically bypassed or hot-swapped.

Part 3. Continuity & Time-Recovery Rubric

Prioritizes instructional continuity and lesson preservation over engineering or technical aesthetics.

Score Metric Operational Threshold Pedagogical Impact Definition
Score 3 Diagnosed & resolved within 120 seconds. Full Resolution / Workaround: $REP_{30} \ge 75\%$ and necessary tool is present in spares inventory. Lesson proceeds completely as planned; zero media modification or student disengagement.
Score 2 Functionality restored via alternative patch. Diminished Capability: Educator recalls steps ($REP_{30} \ge 75\%$) but tool limits force a compromised format or content omission (e.g., continuing via un-remedied loose pins turning a screen blue/green).
Score 1 Hardware break prevents immediate recovery. Minor Mitigation / Accurate Routing: Presentation abandoned due to a Hard Architecture boundary. However, educator accurately pinpoints the failure mode, reducing subsequent technician discovery latency.
Score 0 Persistent fault, complete lockout. Total Abandonment: $REP_{30} < 75\%$. Cognitive trace is too low to resolve the issue under pressure. Technology fatigue forces total environmental abandonment and maximum instructional loss.

Part 5. Analytical Output Generation Formula (CRA-T Format)

Every scenario audit must output a standardized text block utilizing the Competence Retention Analysis Technique (CRA-T) format:

  1. Scenario ID & Target Cohort: Index reference tag.
  2. Pre-Intervention Baseline Outcome: Scored 0-3 using the 10%-20% non-guided intuitive action baseline.
  3. Quantitative Cognitive Modeling ($REP_{30}$ Evaluation): Explicit breakdown of each math variable value applied, the calculated total score percentage, and an indication of whether it passed the 75% retention threshold.
  4. Final Predictive Efficacy Score: Definitive 0-3 grading factoring memory probability and local tool bags, complete with a concise engineering and cognitive justification block identifying curriculum flaws or optimizations.
